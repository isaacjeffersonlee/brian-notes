My Arch Linux Install

Introduction

This is a brief guide for setting up a new arch install with my configs.

Introduction

Idea: Use git and gnu stow to version control the dotfiles all in one place and then symlink them to their respective config directories.

Install Core Packages

Example

First we must install the necessary core packages required to get our dotfiles from github:

xxxxxxxxxx
sudo pacman -Sy ranger stow git zsh kitty foot fd git fzf curl sway swaylock swayidle swaybg wl-clipboard xorg-xwayland

Dotfiles

First clone:

xxxxxxxxxx
git clone https://github.com/isaacjeffersonlee/dotfiles

Ensure we are in the correct directory:

xxxxxxxxxx
cd ~/dotfiles

Then using gnu stow we can stow the individual config directories:

xxxxxxxxxx
stow Config Zsh Scripts

 

Packages

After the core packages have been installed and we have cloned dotfiles and stowed them, we can install the rest of the packages...

Arch Packages

xxxxxxxxxx
sudo pacman -Sy $(cat $HOME/.config/.arch_pkgs.txt)

AUR Packages

xxxxxxxxxx
sudo yay -Sy $(cat $HOME/.config/.aur_pkgs.txt)

 

Change Shell

xxxxxxxxxx
chsh -s "$(which zsh)"

 

Services

Expressvpn

Make sure expressvpn is installed from the aur:

xxxxxxxxxx
yay -S expressvpn

Then enable and start the service:

xxxxxxxxxx
sudo systemctl enable expressvpn && sudo systemctl start expressvpn

 

Udevmon & Interception Tools

First install the required packages:

xxxxxxxxxx
sudo pacman -S interception-tools-dual-function-keys interception-tools

Ensure you're in the correct directory.

xxxxxxxxxx
cd ~/dotfiles/Interception/

Then Symlink the configs to /etc/interception/

xxxxxxxxxx
sudo ln ./udevmon.yaml /etc/interception/
xxxxxxxxxx
sudo ln ./interception.yaml /etc/interception/

Enable and start the services:

xxxxxxxxxx
sudo systemctl enable udevmon && sudo systemctl start udevmon

 

Ly

Ensure the arch package is installed:

xxxxxxxxxx
sudo pacman -S ly

Enable and start the service:

xxxxxxxxxx
sudo systemctl enable ly && sudo systemctl start ly

 

Automatic Backup with crontab

Install cronie:

xxxxxxxxxx
sudo pacman -S cronie

Enable and start the service:

xxxxxxxxxx
sudo systemctl enable cronie.service && sudo systemctl start cronie.service

Edit the user crontab with:

xxxxxxxxxx
crontab -e

And append the following:

xxxxxxxxxx
00 19 * * * /home/isaac/dotfiles/Scripts/.scripts/backup

Thunderbird Setup

ICL Account

For more info see: web src

  1. Enter your name, full College email address e.g., firstname.surname@imperial.ac.uk or initial.surname@imperial.ac.uk, and College password.
  2. Click Continue and wait for the setup to finish. Thunderbird will fail to find the settings on your email account. This is expected.
  3. Enter the following details:
  1. Click Re-test.
  2. In Incoming server authentication method select Oauth2
  3. In Outgoing server authentication method select Oauth2
  4. Click Done to complete the account set up.
  5. You should be prompted to complete the normal web-based two-factor authentication process.
